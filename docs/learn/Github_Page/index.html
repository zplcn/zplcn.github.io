              静态个人网站搭建 #

              简单介绍 #

              零成本搭建个人网站，存放文档与blog，用于知识分享与自我展示，这里我将介绍如何通过Github Page搭建个人网站。

              什么是静态网站 #

              通常我们访问的一些大型网站，往往带有数据的交互，比如你登陆你的个人账号，数据将会发送到服务端，服务端处理后将登陆结构返回给个人电脑。对于静态网页，用于存放一些固定不变的数据，且很少能进行数据交互。

              尽管如此，静态网站也有其优点，也就是其搭建简单，不需要服务器，不需要太多前置知识，适合用于blog与文档分享的场景，让我们能够专注于文档的编写。

              前置准备 #

              如果你准备好了搭建一个静态网站，以下前置工作可能需要进行。

              1. 注册Github账号
              2. 绑定账号和本地
              3. 熟悉使用markdown编写文档
              4. 熟悉hugo

              hugo安装 #

              hugo是一个静态网站生成平台，其根据你的配置、内容，自动生成一个页面。相比于我们手动去编写html文件，无疑更加有效率。

              安装方法 #

              在Macos平台下，推荐使用brew安装:

              brew install hugo

              在Linux平台下，推荐使用apt安装:

              apt install hugo

              创建一个静态网站 #

              使用hugo new site <dirname>即可创建一个静态网站文件夹 首先进入一个文件夹，比如我创建的路径为mydoc

              hugo new site mydoc

              注意如果你使用的是Macos系统，由于其对大小写不敏感，可能导致一些问题，建议新建一个大小写敏感的磁盘用来搭建这个静态网页。

              于是mydoc被创建，其中有以下文件:

              archetypes	content		hugo.toml	layouts		themes
assets		data		i18n		static

              使用git init将当前目录初始化为git目录

              content用于存放我们的markdown文档，archetypes用于存放新建页面的模版，themes用于存放下载的主题，hugo.toml则是网页搭建的初始配置，static存放一些静态数据，比如图片。

              此时打开hugo的 可选主题,在其中选择自己喜欢的主题，这里我选择了book主题，也以此作为案例

              打开book下载页面，提示使用git submodule add https://github.com/alex-shpak/hugo-book themes/hugo-book进行下载 输入后等待一段时间即可完成

              下载完成后，主题存放在themes中 themes/hugo-book/exampleSite是该主题的示例网站，我们可以在其基础上改动 cp -rf themes/hugo-book/exampleSite . 此时执行hugo serve -D即可打开网站的预览,在浏览器上输入127.0.0.1:1313即可查看 /Users/plz/Desktop/截屏2025-02-09\ 21.55.36.png
